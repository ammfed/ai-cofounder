# Context Map

This repo uses a multi-context domain documentation layout.

## Contexts

No application contexts have been defined yet.

When a context is introduced, add an entry here that points to its `CONTEXT.md`.

```text
- <context-name>: src/<context-name>/CONTEXT.md
```

## ADRs

- System-wide ADRs live in `docs/adr/`.
- Context-specific ADRs live in `src/<context-name>/docs/adr/`.
